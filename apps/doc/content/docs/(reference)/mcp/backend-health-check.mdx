  <ReferenceMain>
    <OperationHeader
      tag="MCP Tool"
      title="backend_health_check"
      summary="Check the health endpoint of one backend service."
      method="TOOL"
      path="backend_health_check"
    />

    ## Authorizations

    This tool is read-only. The MCP server itself is reached through `/mcp` over streamable HTTP.

    ## Arguments

    | Name | Type | Required | Description |
    | --- | --- | --- | --- |
    | `base_url` | `string` | Yes | Backend base URL such as `https://api.example.com`. |
    | `service` | `string` | Yes | One of `gateway`, `auth`, `user`, `organizations`, `permission`, `billing`, `notification`. |

    ## Behavior

    The tool resolves the target health URL as `base_url + /health` and performs a GET request
    against that service.

    ## Response

    | Field | Type | Description |
    | --- | --- | --- |
    | `service` | `string` | The requested service key. |
    | `url` | `string` | The health URL used by the tool. |
    | `status_code` | `number` | HTTP status code returned by the target. |
    | `body` | `string` | Serialized JSON health body returned by the service. |
    | `healthy` | `boolean` | `true` when the status code is in the 2xx range. |

    ## Error state

    The tool returns an MCP error result when `base_url` is missing or when `service` is outside
    the supported values.
  </ReferenceMain>
