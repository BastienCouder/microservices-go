<NoteBox title="Transport" tone="primary">
The current MCP server exposes streamable HTTP on `/mcp`. Give that URL directly to any MCP
client that supports remote HTTP transports.
</NoteBox>

## Connection model

<Steps>
  <Step>

### 1. Use the workspace MCP URL

Your MCP endpoint follows the same host as the API gateway:

```text
https://api.example.com/mcp
```

  </Step>
  <Step>

### 2. Register the remote server in your client

```json
{
  "mcpServers": {
    "ai-reco-monitor": {
      "transport": "http",
      "url": "https://api.example.com/mcp"
    }
  }
}
```

  </Step>
  <Step>

### 3. Call the available tools

Use `backend_health_check` to verify backend availability and `backend_list_endpoints` to inspect
the route surface currently exported by the MCP server.

  </Step>
</Steps>

## Tool catalog

<ToolGrid>
  <ToolCard
    name="backend_health_check"
    transport="streamable HTTP"
    input="base_url, service"
    output="service, url, status_code, body, healthy"
  >
    Checks the `/health` endpoint of one backend service. Supported service values:
    `gateway`, `auth`, `user`, `organizations`, `permission`, `billing`, `notification`.
  </ToolCard>

  <ToolCard
    name="backend_list_endpoints"
    transport="streamable HTTP"
    input="none"
    output="base_url, endpoints[]"
  >
    Returns the compact endpoint list currently registered by the MCP server.
  </ToolCard>
</ToolGrid>

<PanelGrid>
  <PanelCard title="backend_health_check" href="/docs/mcp/backend-health-check" kicker="Tool">
    Validate backend availability with a service name and one base URL.
  </PanelCard>

  <PanelCard title="backend_list_endpoints" href="/docs/mcp/backend-list-endpoints" kicker="Tool">
    Return the current compact route catalog exposed by the MCP server.
  </PanelCard>
</PanelGrid>

## Scope of the current MCP surface

The MCP process also exposes `/metrics`, but customer integrations should target `/mcp`.

The current tool catalog is intentionally narrow. For project creation, analysis runs, billing and
attribution, the HTTP API sections in this portal remain the source of truth.
