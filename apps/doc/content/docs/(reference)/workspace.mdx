title: Workspace
description: Projects, prompts, competitors and model configuration.

<NoteBox title="Scope" tone="primary">
Workspace routes are organization-scoped. Send `X-Organization-ID` on every request even though
the gateway can derive organization context for `/projects` when the header is omitted.
</NoteBox>

## Projects

<EndpointTable>
  <EndpointRow method="POST" path="/projects" auth="Session" scope="Organization" response="Success envelope">
    Creates a project. Body fields: `name`, `domain`, `websiteUrl`, `brandName`,
    `brandDescription`, `industry`, `primaryLanguage`, `country`.
  </EndpointRow>

  <EndpointRow method="GET" path="/projects" auth="Session" scope="Organization" response="Success envelope">
    Lists projects for the current organization.
  </EndpointRow>

  <EndpointRow method="GET" path="/projects/{projectId}" auth="Session" scope="Organization" response="Success envelope">
    Reads a single project.
  </EndpointRow>

  <EndpointRow method="PATCH" path="/projects/{projectId}" auth="Session" scope="Organization" response="Success envelope">
    Updates project fields such as `name`, `domain`, `websiteUrl`, `brandName`,
    `brandDescription` or `industry`.
  </EndpointRow>

  <EndpointRow method="POST" path="/projects/{projectId}/activate" auth="Session" scope="Organization" response="Success envelope">
    Moves the project into its active state.
  </EndpointRow>

  <EndpointRow method="POST" path="/projects/{projectId}/finalize" auth="Session" scope="Organization" response="Success envelope">
    Finalizes onboarding for the project and returns the finalization result.
  </EndpointRow>
</EndpointTable>

```bash
curl https://api.example.com/projects \
  --request POST \
  --header "Content-Type: application/json" \
  --header "X-Session-Token: <session-token>" \
  --header "X-Organization-ID: 42" \
  --data '{
    "name": "Acme GEO",
    "domain": "acme.com",
    "websiteUrl": "https://acme.com",
    "brandName": "Acme",
    "brandDescription": "B2B analytics platform",
    "industry": "software",
    "primaryLanguage": "en",
    "country": "US"
  }'
```

## Prompts

<EndpointTable>
  <EndpointRow method="POST" path="/projects/{projectId}/prompts" auth="Session" scope="Organization" response="Success envelope">
    Adds prompts to a project. Body field: `prompts`, an array of prompt strings.
  </EndpointRow>

  <EndpointRow method="GET" path="/projects/{projectId}/prompts" auth="Session" scope="Organization" response="Success envelope">
    Lists prompts for a project. Query params: `search`, `page`, `page_size`.
  </EndpointRow>

  <EndpointRow method="PATCH" path="/projects/{projectId}/prompts/status" auth="Session" scope="Organization" response="Success envelope">
    Updates the status of multiple prompts. Body fields: `promptIds`, `status`.
  </EndpointRow>

  <EndpointRow method="PATCH" path="/prompts/{promptId}" auth="Session" scope="Organization" response="Success envelope">
    Updates a single prompt. Supported fields include `text`, `intent`, `modelIds`, `schedule`,
    `status` and `isActive`.
  </EndpointRow>

  <EndpointRow method="DELETE" path="/prompts/{promptId}" auth="Session" scope="Organization" response="Success envelope">
    Deletes one prompt.
  </EndpointRow>
</EndpointTable>

## Competitors and models

<EndpointTable>
  <EndpointRow method="POST" path="/projects/{projectId}/competitors" auth="Session" scope="Organization" response="Success envelope">
    Adds competitors to a project. Body field: `competitors`.
  </EndpointRow>

  <EndpointRow method="GET" path="/projects/{projectId}/competitors" auth="Session" scope="Organization" response="Success envelope">
    Lists competitors for a project.
  </EndpointRow>

  <EndpointRow method="PATCH" path="/competitors/{competitorId}" auth="Session" scope="Organization" response="Success envelope">
    Updates a competitor. Supported fields: `name`, `domain`, `websiteUrl`, `isActive`.
  </EndpointRow>

  <EndpointRow method="DELETE" path="/competitors/{competitorId}" auth="Session" scope="Organization" response="Success envelope">
    Deletes one competitor.
  </EndpointRow>

  <EndpointRow method="GET" path="/projects/ai-models" auth="Session" response="Success envelope">
    Lists AI models. This route is an alias for the same model catalog exposed at `/ai-models`.
  </EndpointRow>

  <EndpointRow method="POST" path="/projects/ai-models/seed" auth="Session" response="Success envelope">
    Seeds default AI models through the project namespace.
  </EndpointRow>

  <EndpointRow method="GET" path="/ai-models" auth="Session" response="Success envelope">
    Lists AI models. Supports the `active_only` query flag.
  </EndpointRow>

  <EndpointRow method="POST" path="/ai-models/seed" auth="Session" response="Success envelope">
    Seeds default AI models through the top-level model namespace.
  </EndpointRow>

  <EndpointRow method="GET" path="/projects/{projectId}/models" auth="Session" scope="Organization" response="Success envelope">
    Returns the model selection attached to one project.
  </EndpointRow>

  <EndpointRow method="PATCH" path="/projects/{projectId}/models" auth="Session" scope="Organization" response="Success envelope">
    Replaces the project model selection. Body field: `modelIds`.
  </EndpointRow>
</EndpointTable>
