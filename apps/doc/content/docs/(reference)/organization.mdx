<NoteBox title="Workspace scope" tone="primary">
Organizations are the top-level scope for billing, projects and insights. Once a user has joined
an organization, reuse its ID in `X-Organization-ID` for every organization-scoped request.
</NoteBox>

## Organization structure

<EndpointTable>
  <EndpointRow method="POST" path="/organizations" auth="Session" response="Organization object">
    Creates a new organization and attaches the authenticated user as the initial member.
  </EndpointRow>

  <EndpointRow method="GET" path="/organizations/me" auth="Session" response="Membership list">
    Lists the organizations linked to the current user and exposes the effective role for each
    membership.
  </EndpointRow>

  <EndpointRow method="GET" path="/organizations/{id}" auth="Session" scope="Organization permission" response="Organization object">
    Reads one organization by ID.
  </EndpointRow>

  <EndpointRow method="POST" path="/organizations/{id}/teams" auth="Session" scope="Organization permission" response="Team object">
    Creates a team inside the organization. Body field: `name`.
  </EndpointRow>

  <EndpointRow method="GET" path="/organizations/{id}/teams" auth="Session" scope="Organization permission" response="Team list">
    Lists teams for the selected organization.
  </EndpointRow>

  <EndpointRow method="POST" path="/organizations/{id}/members" auth="Session" scope="Organization permission" response="Member object">
    Adds the authenticated user as a member of the target organization.
  </EndpointRow>

  <EndpointRow method="GET" path="/organizations/{id}/members" auth="Session" scope="Organization permission" response="Member list">
    Lists organization members.
  </EndpointRow>

  <EndpointRow method="POST" path="/organizations/{id}/members/{user_id}/roles" auth="Session" scope="Organization permission" response="Member roles">
    Assigns a role to a member. In the current implementation the `{user_id}` must match the
    authenticated user.
  </EndpointRow>
</EndpointTable>

## Invitations and permissions

<EndpointTable>
  <EndpointRow method="POST" path="/organizations/{id}/invitations" auth="Session" scope="Organization permission" response="Invitation object">
    Creates an invitation. Body fields: `email`, `role`, `message`, `expires_at`.
  </EndpointRow>

  <EndpointRow method="GET" path="/organizations/{id}/invitations" auth="Session" scope="Organization permission" response="Invitation list">
    Lists all invitations for the organization.
  </EndpointRow>

  <EndpointRow method="GET" path="/organizations/{id}/invitations/{invitation_id}" auth="Session" scope="Organization permission" response="Invitation object">
    Reads a single invitation.
  </EndpointRow>

  <EndpointRow method="PUT" path="/organizations/{id}/invitations/{invitation_id}" auth="Session" scope="Organization permission" response="Invitation object">
    Updates an invitation with the same fields used at creation time.
  </EndpointRow>

  <EndpointRow method="DELETE" path="/organizations/{id}/invitations/{invitation_id}" auth="Session" scope="Organization permission" response="204 No Content">
    Deletes an invitation.
  </EndpointRow>

  <EndpointRow method="POST" path="/invitations/{token}/accept" auth="Session" response="Invitation plus member data">
    Accepts an invitation token as the currently authenticated user.
  </EndpointRow>

  <EndpointRow method="POST" path="/invitations/{token}/refuse" auth="Session" response="Invitation object">
    Refuses an invitation token as the currently authenticated user.
  </EndpointRow>

  <EndpointRow method="POST" path="/permissions/check" auth="Session" scope="Organization permission" response="Permission decision">
    Evaluates whether the current user can perform an `action` on a `resource` inside the
    organization.
  </EndpointRow>
</EndpointTable>

```bash
curl https://api.example.com/organizations/42/invitations \
  --request POST \
  --header "Content-Type: application/json" \
  --header "X-Session-Token: <session-token>" \
  --header "X-Organization-ID: 42" \
  --data '{
    "email": "new.member@example.com",
    "role": "member",
    "message": "Join our workspace",
    "expires_at": "2026-04-01T00:00:00Z"
  }'
```

## Billing and quota management

<EndpointTable>
  <EndpointRow method="POST" path="/billing/subscriptions" auth="Session" scope="Organization" response="Subscription object">
    Creates or updates the subscription state for an organization. Body fields:
    `organization_id`, `plan`, `seats`, `monthly_quota`.
  </EndpointRow>

  <EndpointRow method="GET" path="/billing/quotas/{organization_id}" auth="Session" scope="Organization" response="Quota summary">
    Returns the current plan, monthly quota and seats for one organization.
  </EndpointRow>

  <EndpointRow method="POST" path="/billing/stripe/checkout-session" auth="Session" scope="Organization" response="Stripe checkout session">
    Builds a Stripe checkout session. Body fields include `plan`, `billing_cycle`, `seats`,
    `correction_credits`, `success_url`, `cancel_url` and optional `request_id`.
  </EndpointRow>

  <EndpointRow method="POST" path="/billing/stripe/customer-portal" auth="Session" scope="Organization" response="Stripe portal session">
    Opens a Stripe customer portal session for the active organization.
  </EndpointRow>

  <EndpointRow method="POST" path="/billing/stripe/webhook" auth="None" response="Status ok payload">
    Stripe webhook callback. This route is intentionally reachable without a user session because
    it is server-to-server.
  </EndpointRow>
</EndpointTable>

## Operational notifications

<EndpointTable>
  <EndpointRow method="POST" path="/notifications/send" auth="Session" response="Accepted notification">
    Sends a notification through the selected `channel`. Body fields: `channel`, `recipient`,
    `subject`, `message`.
  </EndpointRow>

  <EndpointRow method="GET" path="/notifications" auth="Session" response="Notification list">
    Lists recent notifications. Use the `limit` query parameter.
  </EndpointRow>
</EndpointTable>

```bash
curl https://api.example.com/billing/stripe/checkout-session \
  --request POST \
  --header "Content-Type: application/json" \
  --header "X-Session-Token: <session-token>" \
  --header "X-Organization-ID: 42" \
  --data '{
    "organization_id": 42,
    "plan": "pro",
    "billing_cycle": "monthly",
    "seats": 5,
    "correction_credits": 250,
    "success_url": "https://app.example.com/billing/success",
    "cancel_url": "https://app.example.com/billing/cancel"
  }'
```
