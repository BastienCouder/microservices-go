## Integration path

<Steps>
  <Step>

### 1. Open a session

Start with `/auth/password`, `/auth/otp/start` plus `/auth/otp/verify`, or `/auth/oidc/google`.
Then validate the session through `/auth/validate`.

  </Step>
  <Step>

### 2. Bootstrap the user and organization

Create the user profile with `/users`, read `/users/me`, then create or pick an organization from
`/organizations` and `/organizations/me`.

  </Step>
  <Step>

### 3. Use business routes

Continue with the business sections in this portal: organization admin, workspace setup, insights
and automation.

  </Step>
  <Step>

### 4. Connect MCP clients

Point your MCP client to `/mcp`, then call `backend_health_check` and
`backend_list_endpoints`.

  </Step>
</Steps>

## Coverage

Public gateway routes are grouped by customer workflow:

- `/auth` in [Authentication](/docs/authentication)
- `/users` and `/admin/users` in [Profile](/docs/profile)
- `/organizations`, `/invitations`, `/permissions`, `/billing` and `/notifications` in
  [Organization](/docs/organization)
- `/projects`, `/prompts`, `/competitors` and `/ai-models` in [Workspace](/docs/workspace)
- `/analysis`, `/ai` and `/attribution` in [Insights](/docs/insights)
- `/mcp` in [MCP](/docs/mcp)
